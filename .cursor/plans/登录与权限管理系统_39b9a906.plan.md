## 目标与范围
- **目标**：交付一套可用于后台管理系统的“登录 + 权限管理”基础底座，支持用户登录、会话续期、角色授权、前端路由/按钮控制、后端接口鉴权、审计追踪。
- **不在本期**（可作为二期）：ABAC（基于属性的策略）、复杂数据权限（行/列级）、组织架构/部门树、SSO（OIDC/SAML）、多租户隔离。

## 业务/产品分析（PRD 级）
### 角色与使用场景
- **SuperAdmin**：系统初始化、创建权限与菜单、创建角色、分配角色给用户。
- **Admin**：日常用户管理、角色分配、查看审计日志。
- **User**：登录后仅能访问被授权的菜单与功能。

### 权限粒度（建议默认）
- **API 权限**（后端强制）：以 `permission_code` 表示，如 `user:read`, `user:create`, `role:assign`。
- **前端功能点权限**（按钮/操作级）：复用同一 `permission_code` 控制按钮显示与操作可用。
- **菜单/路由权限**（可选强关联）：菜单项关联一组 `permission_code`，用于决定路由可见性。

### 核心用户旅程
1. **登录**：用户名/密码 → 返回 `access_token`（短期）+ `refresh_token`（长期）+ 用户基本信息 + 权限摘要。
2. **访问**：前端请求携带 `Authorization: Bearer <access>` → 后端鉴权与授权。
3. **续期**：`access_token` 过期 → 用 `refresh_token` 换新 `access_token`（可旋转 refresh）。
4. **登出**：前端清 token；后端可选把 refresh 加入黑名单/吊销。

### 管理端功能（最小可用集）
- **登录页**：账号/密码、记住我、错误提示、强制改密（可选）。
- **仪表盘**：展示当前用户、角色、最近登录信息（可选）。
- **用户管理**：列表/搜索/新增/禁用/重置密码/分配角色。
- **角色管理**：增删改查、为角色勾选权限。
- **权限管理**：权限字典（code/name/description）、（可选）分组。
- **菜单管理（可选但推荐）**：菜单树、绑定路由、配置可见权限。
- **审计日志（推荐）**：登录日志、关键操作日志。

### 关键异常与安全要求
- **错误信息**：登录失败不泄露“用户不存在/密码错误”细节。
- **密码策略**：BCrypt；可配置最小长度/复杂度；密码重置流程（管理端重置为临时密码或强制改密）。
- **令牌安全**：refresh_token 建议存 httpOnly cookie（更安全）或本地存储（实现更简单但风险更高）；需要明确选型。
- **权限变更生效**：权限/角色变更后，已登录用户何时生效（建议：刷新 token 时生效；或强制用户重新登录）。
- **审计**：记录操作者、目标对象、结果、时间、IP、UA。

## 技术方案（Spring Boot + Vue）
### 后端架构
- **认证**：Spring Security + JWT
  - `access_token`：短有效期（如 15min）
  - `refresh_token`：长有效期（如 7-30d），建议旋转 + 服务端存储/黑名单
- **授权**：方法级注解 `@PreAuthorize("hasAuthority('user:read')")` 或基于过滤器的 URL-权限映射
- **缓存（可选）**：Redis 用于 refresh token 存储、黑名单、权限缓存

### 数据持久化（MySQL，生产/部署）
- **结论**：可以使用现有 **MySQL** 环境作为运行库；Spring Boot 2.7 + JPA + Flyway 均官方支持 MySQL。
- **依赖**：在 [`backend/pom.xml`](backend/pom.xml) 使用 `com.mysql:mysql-connector-j`（runtime）；**仅支持 MySQL 5.7**，不再引入 H2 等其他数据库驱动。
- **配置**：为生产增加 `application-prod.yml`（或等价外部配置），设置例如：
  - `spring.datasource.url=jdbc:mysql://<host>:3306/<db>?useSSL=...&serverTimezone=...&characterEncoding=utf8`（参数按环境与安全要求选择）
  - `spring.datasource.username` / `password`
  - `spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQL57Dialect`（当前环境为 **MySQL 5.7**；若为 8.0 可改为 `MySQL8Dialect`）
- **Flyway 与 DDL**：[`backend/src/main/resources/db/migration/V1__init.sql`](backend/src/main/resources/db/migration/V1__init.sql) 为 **MySQL 5.7** 语法（`AUTO_INCREMENT`、`DATETIME(6)`、`InnoDB`/`utf8mb4` 等）；仅面向该方言维护。
- **初始化数据**：若存在 [`DataInitializer`](backend/src/main/java/com/example/authz/bootstrap/DataInitializer.java) 等种子数据，需在 MySQL 上同样验证编码与大小写规则（表名是否大小写敏感取决于 OS/`lower_case_table_names`）。
- **验收**：使用 MySQL 作为唯一运行库时，登录、RBAC、审计、refresh 旋转全链路通过；`mvn test` 使用 **`test` profile** 连接本机 MySQL 5.7（默认库名 `authz_test`，可通过 `MYSQL_TEST_*` 环境变量覆盖），需预先建库；不引入 H2/Testcontainers。

### 前端架构（Vue）
- **状态管理**：Pinia（推荐）
- **路由守卫**：
  - 未登录 → 跳转登录
  - 已登录但无路由权限 → 403 页面或重定向
- **权限指令**：如 `v-permission="'user:create'"` 控制按钮显示/禁用
- **请求拦截**：axios 拦截器自动附带 access token；401 时尝试 refresh；失败则回登录

## 数据模型（建议表）
- `sys_user(id, username, password_hash, status, created_at, updated_at, last_login_at, ...)`
- `sys_role(id, code, name, description, ...)`
- `sys_permission(id, code, name, description, ...)`
- `sys_user_role(user_id, role_id)`
- `sys_role_permission(role_id, permission_id)`
- `sys_menu(id, parent_id, name, path, component, icon, sort, visible, ...)`（可选）
- `sys_menu_permission(menu_id, permission_id)`（可选）
- `sys_refresh_token(id, user_id, token_hash, expires_at, revoked_at, created_at, device, ip, ua)`（推荐）
- `sys_audit_log(id, actor_user_id, action, target_type, target_id, result, detail, ip, ua, created_at)`（推荐）

## API（建议最小集合）
### Auth
- `POST /api/auth/login`：返回 access/refresh + profile + permissions
- `POST /api/auth/refresh`：refresh 换新 access（可旋转 refresh）
- `POST /api/auth/logout`：吊销 refresh（可选）
- `GET /api/auth/me`：当前用户信息 + 权限

### Users/Roles/Permissions
- `GET/POST/PUT/DELETE /api/users`
- `POST /api/users/{id}/roles`（分配角色）
- `GET/POST/PUT/DELETE /api/roles`
- `POST /api/roles/{id}/permissions`（勾选权限）
- `GET/POST/PUT/DELETE /api/permissions`

### Menus（可选）
- `GET/POST/PUT/DELETE /api/menus`
- `GET /api/menus/tree`

### Audit（推荐）
- `GET /api/audit-logs`

## 验收标准（MVP）
- 能登录/退出/自动续期，401/403 行为正确
- 后端接口在无 token / 无权限时分别返回 401/403
- 前端路由与按钮在无权限时不可见或不可用
- 管理端可创建用户、角色、权限并完成分配
- 关键操作与登录行为可在审计日志查询到

## 关键待定（会影响实现）
- refresh_token 存储：**httpOnly cookie（更安全）** vs **localStorage（更简单）**
- 权限下发策略：登录一次返回全量 vs `GET /me` 拉取 vs 按需拉取
- 菜单是否作为一等公民（决定是否需要菜单表与动态路由）

## 里程碑建议
- **M1（基础鉴权）**：登录/refresh/logout + Spring Security JWT + axios 拦截器
- **M2（RBAC）**：用户/角色/权限表 + 管理端 CRUD + 接口鉴权
- **M3（前端权限）**：动态路由/菜单、按钮权限指令、403 页面
- **M4（可运营）**：审计日志、密码重置与安全策略、可观测性与告警（可选）
